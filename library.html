    This page discusses the structure of the core LibPKMN library.

    Abstractions

    The key to LibPKMN is abstraction, whether between different game save formats or simply between Pokémon of different generations.

    There have been many changes to the games between generations. Different attributes were added, certain calculations were done differently, and some stats changed altogether. However, the user does not need to worry about any of this. In processing parameters passed by the users, each class's factory function deals with this, and the user uses a shared pointer to the correct implementation. For example, if the user calls for an Eevee Base Pokemon from a Generation I game and asks for its evolutions, it will receive a list with Vaporeon, Jolteon, and Flareon, the only evolutions present in these games. However, an Eevee Base Pokemon from a Generation IV game will return a list that also includes Espeon, Umbreon, Glaceon, and Leafeon.

    This paradigm becomes even more prevalent in the Game Save class, which interacts directly with a binary game save. All six generations store the game data in vastly different ways, but with LibPKMN, the user does not deal with this at all. For instance, let's say the user wants to edit the trainer's team. Normally, they would need to know where in the binary file the party is stored and what kind of parsing and bit-shifting is necessary to access different attributes. These methods vary wildly between generations, but with LibPKMN, this is all that's necessary:

    pkmn::game_save::sptr sav = pkmn::game_save::make("pokemon_emerald.sav");
pkmn::pokemon_team_t team;
sav->get_party(team);

    And now the user has a vector of easy editable Pokémon. Once they're done, they simply need to call:

    sav->save();

    to save their work and get back to playing.

    Underlying SQLite database

    Underneath the library lies a SQLite database, containing vast amounts of information on every aspect of the Pokémon games. This database can be found here. For the most part, the database remains the same as what is given in the link, with only minor modifications for some missing Generation I-II values. All stats, names, and other values relevant to each class are queried from this database.

    This database is automatically compiled from a dump and installed on the user's system alongside LibPKMN.
